MAGUIRE BLENDER INTEGRATION

We speak fluent Maguire.

Maguire gravimetric weigh-batch blenders are workhorses — but the good data lives behind a proprietary protocol, and the controller only shows it two lines at a time on a keypad LCD. We pull it out. Real consumption, real batch weights, real cost-per-shift — off the actual hardware, onto a dashboard, and into your ERP. No guessing, no "the vendor says it can't be done."

The data is in there. Getting it out is the job.

A Maguire's Modbus TCP port isn't a normal Modbus device — it's a wrapper around Maguire's own MLAN protocol. Point standard register tools at it and you get noise. We mapped the real path: MLAN command codes (Get Totals, Cycle Weight, Batch Info) tunneled inside the Modbus packets, plus a careful parse of the keypad LCD buffer for the fields the protocol won't hand you directly. The result is honest numbers — and where the firmware genuinely can't report a value live, we label it a snapshot with a timestamp instead of faking it.

WHAT WE DELIVER

The Maguire toolkit.

Live operator dashboard

Glanceable per-machine screens: recipe, component setpoints, live/last-reported weight, batch state. Runs in demo mode or live mode, so operators can train on it before it ever touches the real controller.

💵

Cost analytics

Per-hopper material cost computed on real consumption deltas — mid-shift price changes costed correctly, before and after. Editable bill-of-materials with supplier and price per component.

📈

Shift & production reporting

Clean 8-hour shift rollups for accounting and purchasing: day / month / year filters, per-drop batch logging with gap detection (a missed drop is shown, never fabricated), one-click PDF, Excel, and CSV.

🔧

Blender troubleshooter

Decodes what the controller is actually doing — component types, setpoints, batch targets — and flags the states that trip crews up. Built from the register / MLAN behavior we reverse-engineered.

🔗

eIQ / DELMIAWorks hand-off

Material consumption pushed into eIQ the way eIQ actually wants it: BOM-driven backflush against a work order. Every inbound method covered — folder-watch CSV/XML, WebAPI with token auth, FTPS, live Modbus/OPC UA/MQTT, MTConnect. One config file is the only thing your IT ever edits.

🧰

Commissioning tools

Field setup without the vendor on the phone: a preflight validator that catches the silent killers (missing unit-of-measure factors, empty part-number crosswalks) before go-live, plus the controller-side sequence to actually enable the data feed. We've done the fussy parts already.

BEYOND MAGUIRE

This isn't a one-off. We live on the plant floor.

🗺️

Machine troubleshooting portals

We build private, plant-dedicated troubleshooting portals — one home screen, every machine on the floor behind it. Pick a line and you get its troubleshooter and its PLC configuration side by side, the same shape whether the line runs a WAGO PLC or a Yaskawa one. A technician's tool, not a brochure — deployed for a single plant, not sold off a shelf.

🩺

Deep motion-machine troubleshooter

For complex motion machines we go deeper: fault search, a full I/O map, a live PLC monitor over Modbus, the electrical schematic on-screen, and a knowledge base built straight from the machine's ladder program. We've found a marginal braking resistor by reading the run-enable edge and brake handshake — not just reciting an alarm number.

📐

Electrical print generation (JIC)

Letter-landscape drawings with a proper zone border, title block, BOM, and notes — drawn with standard JIC ladder symbols (N.O./N.C. contacts, pushbuttons, coils, beacons, PSU boxes, wire hops). Give us the control scheme; we hand back a client-ready print package with matching operation and sensor guides.

🔍

PLC reverse-engineering

Inherited a machine with 1,000+ steps of uncommented ladder? We monitor the live logic while operating the machine, decode each bit / timer / register, and round-trip the labels back into the project's device comments — so your maintenance team inherits a documented program.

📚

Plant-knowledge AI search

A local semantic search engine across ~1,000 plant PDFs — PLCs, drives, heat controllers, meters. Ask in plain English, get the exact manual and page number back. Runs entirely on-prem; nothing leaves the building.

🛠️

Signed, licensed installers

We don't hand you a folder of scripts. Our industrial software ships as a real Windows installer with our own Ed25519 license keys and code-signing — the same productization discipline as any commercial vendor, with the integration options kept free and open.

TRAINING

We also teach the floor.

The same engineering goes into training. We build interactive electrical training — live, animated schematics driven by a real circuit solver, not slideshow pictures. Relays energize, contacts flip, seal-in latches hold, current lights up the wire. Learners work from real component photos, cutaways, and manufacturer wiring diagrams — the actual hardware, not a prettier cartoon of it — through a baby-steps-to-advanced curriculum. Built for new technicians who need to understand the cabinet, not just memorize it.
